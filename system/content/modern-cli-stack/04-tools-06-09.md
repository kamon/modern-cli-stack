# The 13 Tools (continued)

## 6. ripgrep (`rg`) — Fast Code Search

**Search code across large projects in milliseconds.**

Grep replacement that's dramatically faster, respects `.gitignore`, and skips
binary files automatically.

### Install

```bash
# macOS
brew install ripgrep

# Linux / WSL
sudo apt install ripgrep
```

### Before / After

- **Before:** `grep -r "TODO" .` — slow, spams binary files, includes
  `node_modules`.
- **After:** `rg "TODO"` — instant, respects `.gitignore`, syntax-highlighted.

### Common flags

| Command | What it does |
|---|---|
| `rg -i "error"` | case-insensitive |
| `rg "TODO" --type py` | only Python files |
| `rg "fixme" -l` | files only, no content |
| `rg "auth" -C 3` | 3 lines of context |

**Try this:** `rg "TODO|FIXME" --type-add 'web:*.{html,css,js}' -t web` in
any project.

## 7. fd — Friendly `find`

**`find` syntax that fits on one line.**

Simple, fast, user-friendly alternative to `find`. Smart defaults: ignores
hidden files, respects `.gitignore`, parallel by default.

### Install

```bash
# macOS
brew install fd

# Linux / WSL
sudo apt install fd-find    # script auto-creates `~/.local/bin/fd` symlink
```

### Before / After

- **Before:** `find . -name "*.py" -not -path "*/node_modules/*"`
- **After:** `fd "\.py$"`

**Try this:** `fd config`, `fd -e md` (all markdown), `fd -H "^\.env"`
(hidden env files).

## 8. bat — `cat` With Wings

**Syntax-highlighted, line-numbered, Git-aware `cat`.**

Drop-in `cat` replacement that shows line numbers, syntax highlights, and
integrates with Git to show diff markers.

### Install

```bash
# macOS
brew install bat

# Linux / WSL
sudo apt install bat        # script auto-creates `~/.local/bin/bat` symlink
```

### Before / After

- **Before:** `cat package.json` — wall of text.
- **After:** `bat package.json` — syntax-highlighted, numbered, with `|##|`
  markers for unstaged changes.

**Try this:** `bat --list-languages | head -20` to see supported syntaxes.
Use `bat -A file.sh` to show non-printing chars.

## 9. eza — Modern `ls`

**`ls`, but pretty and informative.**

Drop-in `ls` replacement with sensible defaults: Git status, file icons (with
Nerd Font), tree view, color-coded by file type.

### Install

```bash
# macOS
brew install eza

# Linux / WSL
sudo apt install eza        # Debian 13+ / Ubuntu 24.04+
```

Add to `~/.bashrc`:

```bash
alias ls="eza --icons"
alias ll="eza -la --icons --git"
alias lt="eza --tree --level=2 --icons"
```

### Before / After

- **Before:** `ls -la` — 8 columns of grey text.
- **After:** `ll` — colored, icon-prefixed, with Git status indicators.

**Try this:** `lt` in any project for a 2-level tree. `ll --sort=modified`
for recently-changed files.
